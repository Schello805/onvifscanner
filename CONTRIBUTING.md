# Contributing

Danke, dass du helfen möchtest!

## Regeln

- Bitte keine Features, die Missbrauch erleichtern (z. B. “Internetweite Scans”, Exploits, Default-Passwort-Bruteforce).
- Fokus: Lokal, sicher, nachvollziehbar.
- Keine sensiblen Daten (Credentials, echte IPs, Logfiles) in PRs.

## Development

```bash
npm install
npm run dev
```

## Pull Requests

- Kleine, fokussierte PRs bevorzugt
- Beschreibe “Warum” + “Wie getestet”
